# Plan 01-01: WooClient ŌĆö HTTP Foundation + Cart-Token Session

## Objective

Build the `WooClient` class in `@arc/core`: a framework-agnostic WC Store API v1 HTTP client with transparent Cart-Token session handling, Nonce auto-refresh on `rest_cookie_invalid_nonce`, and hand-authored TypeScript types for the cart + error responses. This is the mandatory pre-step before the 6-agent parallel swarm in plans 02ŌĆō07.

## Context

- ADR-0006 (`arc_cart_token` cookie spec) defines the Cart-Token lifecycle ŌĆö `WooClient` fires `onCartToken` callback; framework layer (`@arc/next`) sets the cookie.
- WC Store API v1 base path: `/wp-json/wc/store/v1`
- Auth model: Cart-Token header (`Cart-Token: <value>`) ŌĆö no nonce required for anonymous cart ops (per ADR-0006 + WC docs)
- Error shape: `{ code: string; message: string; data: { status: number } }`
- Retry: on `rest_cookie_invalid_nonce` ŌĆö refresh nonce, retry exactly once
- Retry: exponential backoff (1s, 2s, 4s) on 5xx, max 3 attempts
- No external HTTP deps ŌĆö native `fetch` only (per STACK ┬¦1.2)

### Task 3: `WooClient` class (`src/client/WooClient.ts`)
**Type:** auto

Implement `WooClient`:

```
class WooClient {
  constructor(options: WooClientOptions)

  // Core request method
  request<T>(path: string, init?: RequestInit & WooRequestOptions): Promise<T>

  // Cart operations
  getCart(): Promise<WooCart>
  addToCart(productId: number, quantity: number): Promise<WooCart>
  updateCartItem(key: string, quantity: number): Promise<WooCart>
  removeCartItem(key: string): Promise<WooCart>
  applyCoupon(code: string): Promise<WooCart>
  removeCoupon(code: string): Promise<WooCart>
}
```

Key behaviours:
- `request()` injects `Cart-Token` header if `getCartToken()` returns a value
- `request()` extracts `Cart-Token` response header, calls `onCartToken` if different from current
- On `WooError` with `code === 'rest_cookie_invalid_nonce'`: refresh nonce via `getNonce?.()`, retry once
- AbortSignal timeout: default 10s, configurable via `WooClientOptions.timeout`
- All cart methods POST/DELETE to correct Store API paths

**Done when:** Unit tests covering token extraction, nonce retry, and cart operations pass.
