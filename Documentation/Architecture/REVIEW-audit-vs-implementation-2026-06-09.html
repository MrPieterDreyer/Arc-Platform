The build is faithful to the plan.

Phases 0–4b shipped what their plans promised: @arc/core (Cart-Token lifecycle, typed Store API + WPGraphQL clients, discriminated-union errors), @arc/next (RSC loaders on 'use cache', HttpOnly cookie bridge, HMAC revalidation), @weave/react / @weave/next (15 input types, RSC-safe SectionRenderer, draft preview), and the WP plugin + Admin editor. 250+ tests, all CI gates green, pnpm build exits 0. The audit found no case where the implementation diverged from an accepted ADR or phase plan.

What the audit found instead: scope the original roadmap never covered.

Customer-auth implementation (ADR-0009 was accepted as a spike, then parked for "Phase 5"), security headers/CSP, and supply-chain release integrity were never phase deliverables. They are now the gating items between Phase 4b and a publishable v0.1 — hence the recommended Phase 4c: Security & Release Hardening below.

3 · Strengths Confirmed

Where Arc/Weave already meets or beats the Hydrogen/Weaverse baseline — verified in both the audit and Phase 0–4b verification reports.

  • Typed discriminated-union error model (api/network/parse) vs Hydrogen's flat GraphQL error array.
  • Runtime Zod validation of cart responses with warn-not-throw drift detection.
  • HMAC-SHA256 webhook revalidation — ±5-min replay window, tag-prefix allowlist, byte-parity Node↔PHP (raw received bytes, never re-serialized).
  • Single require_cap() REST choke-point with an explicit 401/403 auth-matrix test; zero __return_true permission callbacks.
  • Write-safety in the WP plugin — server-minted section IDs, 1 MB body gate, strict unknown-key rejection.
  • Zero unstable_cache usage — clean Next.js 16 'use cache' + cacheTag discipline; loaders construct clients inside the cached fn.
  • RSC-safe SDK — @weave/react barrel omits client-only exports; SectionRenderer never throws.
  • Deploy-target agnostic — pure Next.js, no Oxygen-style runtime coupling.
  • Dual-gate boundary enforcement — next/* imports blocked by Biome and a bash grep failsafe in CI.
